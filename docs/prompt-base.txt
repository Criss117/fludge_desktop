**Contexto del proyecto**

Estoy desarrollando una aplicación POS (punto de venta) desktop con **Wails v2** usando **Go** en el backend y **React/TypeScript** en el frontend. La base de datos es **SQLite** con **sqlc** para las queries. La app es local/offline y multitenant.

---

**Stack técnico**
- Wails v2 (desktop app, Linux, webkit2_41)
- Go con arquitectura DDD (Domain Driven Design)
- SQLite + sqlc (queries escritas a mano)
- React/TypeScript con Zod para validaciones en el frontend
- Wails bindings exponen handlers directamente a React

---

**Arquitectura general**

5 bounded contexts:
Identity & Access     → upstream, todos dependen de él
Catálogo de productos → upstream de ventas e inventario
Gestión de inventario → consumidor de catálogo y ventas
Punto de venta        → emite eventos a inventario y crédito
Crédito & cobranza    → consumidor de ventas

Shared Kernel: `organization_id` presente en todos los agregados como clave de tenancy.

---

**Patrones establecidos globalmente**

- `NewX()` — constructor con validación, retorna `(*X, error)`
- `ReconstituteX()` — rehidrata desde DB sin revalidar, retorna `*X`
- Agregados con **campos públicos** — sin `PrimitiveX`, el mapper accede directo
- VOs con `NewX()` para input externo y `XFromStorage()` para rehidratar desde DB
- `commands/` — DTOs de entrada para use cases (escritura)
- `responses/` — DTOs de salida compartidos entre use cases y queries
- `usecases/` — escrituras, usan agregados, protegen invariantes
- `queries/` — lecturas directas a sqlc, retornan responses, nunca tocan agregados
- `derrors/` — todos los errores de dominio en un solo lugar por contexto
- `ports/repositories.go` — `Create`, `Update`, `Delete`, `FindByX`
- `ports/queries.go` — interfaces de lectura que retornan responses
- `infrastructure/persistence/mapper.go` — único lugar donde sqlc models se convierten a agregados
- CQRS light: agregados para escrituras, queries sqlc directas para lecturas de UI
- Los use cases reciben `ctx context.Context` como primer parámetro, nunca lo guardan en el struct
- El ctx de Wails se guarda en el handler — no se usa `context.Background()`
- Los repositorios reciben `ctx` como parámetro en cada método

**Manejo de sesión:**
- `GetSessionFunc func() *iamAggregates.AppState` — función que cada handler recibe para leer el estado actual
- Cada handler tiene `activeOrganizationID() (string, error)` como helper privado
- Errores de sesión en `internal/shared/derrors/session_errors.go`

**Bounded Context IAM — detalle completo**

**Aggregate roots:**

`Operator` (campos públicos)
- `ID` ulid, `Email` VO, `Pin` VO, `Username`, `Root bool`
- Métodos: `IsRoot()`, `IsActive()`, `VerifyPIN(pin)`, `Delete()`
- `NewOperator()` valida VOs · `ReconstituteOperator()` rehidrata sin revalidar
- Pin nunca se expone en responses

`Organization` (campos públicos)
- `ID`, `Slug` VO, `ContactEmail *VO`, `Logo *string`, `ContactPhone *string`, `Members []*Member`, `Teams []*Team`
- Crea `DefaultTeam` con `AllPermissions()` en `NewOrganization`
- Métodos: `AddMember()`, `RemoveMember()`, `AddTeam()`, `AssignOperatorToTeam()`, `FindMemberByOperatorID()`, `FindTeamByID()`, `FindTeamsByOperatorID()`
- `AssignOperatorToTeam` valida que el operator sea miembro antes de asignar al team

`AppState` (entidad singleton, campos públicos)
- `ActiveOperatorID *string`, `ActiveOrganizationID *string`
- Métodos: `ActivateOperator(id)` limpia org activa · `ActivateOrganization(orgID, org)` valida membresía · `Clear()`
- Solo IDs — no carga agregados completos

**Entidades hijo de Organization:**

`Member` — `ID`, `OperatorID`, `OrganizationID`, `Role` VO MemberRole (ROOT|MEMBER)
`Team` — `ID`, `Name`, `OrganizationID`, `Permissions []Permission`, `Description *string`, `Members []*TeamMember`. DefaultTeam con AllPermissions() al fundar org
`TeamMember` — `ID`, `TeamID`, `OperatorID`, `OrganizationID`. `NewTeamMember()` no retorna error

**Value Objects IAM:**

`Email` — en shared/valueobjects, compartido con catálogo
`Pin` — hasheado, `Matches(other Pin) bool`
`Slug` — lowercase, sin espacios
`MemberRole` — `var MemberRoleRoot = MemberRole{value: "ROOT"}`, receptor valor
`Permission` — `action:resource`. Actions: create/read/update/delete. Resources: team/product/ticket/customer/category/supplier/employee
`PermissionList` — `NewPermissionList()` valida · `PermissionListFromStorage()` confía · `PermissionListFromJSON(json.RawMessage)` para DB · `ToJSON()` para persistir · `Has()` · `HasAll()` · `HasSome()`

**Domain Services (interfaces en ports/services.go):**
`PermissionResolver` — `EffectivePermissions(operatorID, orgID)` · `Can(operatorID, orgID, Permission)`
`MembershipChecker` — `IsMember(operatorID, orgID) bool`
`OrganizationBootstrap` — operación atómica: org + member ROOT + DefaultTeam + asigna founder

**Ports IAM:**

**Use cases IAM implementados:**
- `SignIn` — Username + PIN · activa operator en AppState · deja sin org activa
- `SignOut` — limpia AppState completamente
- `SwitchOrg` — lee AppState para obtener operator activo · valida membresía · retorna SwitchOrgResult{Operator, Organization, Member, Teams}

**SessionState (en appstate/, no en IAM):**
```go
type ActiveOperator struct {
    *iamAggregates.Operator
    *iamAggregates.Member
    Teams []*iamAggregates.Team
}
type SessionState struct {
    ActiveOrganization *iamAggregates.Organization
    ActiveOperator     *ActiveOperator
}
```
Métodos: `IsAuthenticated()` · `HasOrganization()` · `IsRootOperator()` · `IsOrgRoot()` · `Can(Permission)` · `CanAll()` · `CanSome()`

SessionState vive en App con sync.RWMutex. Se actualiza sincrónicamente via callback StateChangeEvent. Separado de AppState (dominio).

**StateChangeEvent:**

**Bounded Context Catálogo — detalle completo**

**Tres aggregate roots independientes** — se referencian solo por ID, nunca se embeben:

`Product` (campos públicos)
- `ID`, `SKU` VO, `Name`, `Description *string`
- `WholesalePrice` VO Money, `SalePrice` VO Money, `CostPrice` VO Money
- `Stock int64`, `MinStock int64` (0 = sin alerta de reorden)
- `CategoryID *string`, `SupplierID *string` — referencias simples opcionales
- `OrganizationID string`
- Invariantes: `sale_price >= cost_price` · `wholesale_price <= sale_price` · `stock >= 0`
- `MinStock < 0` se normaliza a 0 silenciosamente
- `NeedsReorder()` retorna false si MinStock == 0
- Métodos: `UpdatePrices()` · `UpdateMinStock()` · `Delete()` · `IsActive()` · `Equals()`

`Category` (campos públicos)
- `ID`, `Name`, `Description *string`, `OrganizationID`
- Invariante: nombre mínimo 2 caracteres, único por organización
- Métodos: `UpdateName()` · `UpdateDescription()` · `Delete()` · `IsActive()` · `Equals()`

`Supplier` (campos públicos)
- `ID`, `Name`, `ContactPhone *string`, `ContactEmail *VO Email`, `OrganizationID`
- ContactEmail nullable — usa Email VO de shared/valueobjects
- Nombre único por organización
- Métodos: `Delete()` · `IsActive()` · `Equals()`

**Value Objects Catálogo:**

`SKU` — uppercase, no vacío · `NewSKU()` valida · `SKUFromStorage()` confía
`Money` — int64 en centavos, no negativo · `PriceSet` agrupa las tres invariantes de precio juntas

**Ports Catálogo:**

**Use cases Catálogo implementados:**
- `CreateProduct` — valida SKU único · valida nombre único · verifica que categoryID y supplierID existan si se proporcionan
- `FindAllProducts` — query directa a sqlc

**Handler Catálogo:**
```go
type GetSessionFunc func() *iamAggregates.AppState

type CatalogHandler struct {
    ctx               context.Context
    productRepository ports.ProductRepository
    getSession        GetSessionFunc
}

// Helper privado usado en todos los métodos
func (h *CatalogHandler) activeOrganizationID() (string, error) {
    appState := h.getSession()
    if appState == nil {
        return "", shared_derrors.ErrAppStateNotFound
    }
    if appState.ActiveOrganizationID == nil {
        return "", shared_derrors.ErrNoActiveOrganization
    }
    return *appState.ActiveOrganizationID, nil
}
```

**App struct:**

type App struct {

a.CatalogHandler = *catalog.NewCatalogHandler(ctx, queries, func() *iamAggregates.AppState {

    defer a.mu.RUnlock()
    return a.SessionState
})
